Expand description

Repository operations for the goal extension table.

Implementations should store goal-specific rows beside, not inside, the canonical TaskRecord. create_goal exists only to preserve the atomic transaction boundary between the generic task row and its goal extension.

where Self: 'async_trait, 'life0: 'async_trait,

Atomically insert the canonical task row and its goal extension row.

The caller supplies both records because they are different sources of truth: TaskRecord owns lifecycle/routing/ownership, while GoalTaskRecord owns objective/limits/pause details. Implementations must reject mismatched ids or non-goal task kinds rather than repairing them silently.
